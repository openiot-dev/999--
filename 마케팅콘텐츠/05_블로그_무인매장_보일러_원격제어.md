# 무인 파티룸 사장님, 아직도 새벽에 보일러 켜러 매장 가세요?

## 제목 후보 5개

1. **무인 파티룸 사장님, 아직도 새벽에 보일러 켜러 매장 가세요? (무인매장 난방 자동화 총정리)** ← 역공감
2. **보일러 앱 깔았는데 왜 더 불편하죠? 무인매장 난방 원격제어, 전용 컨트롤러 vs 시하스 끝장 비교**
3. **겨울 파티룸, 이거 모르고 보일러 켜두면 빈 방 난방비만 한 달 내내 샙니다**
4. **보일러 회사 앱 믿고 맡겼다가 서비스 종료… 무인매장은 시하스 + 오토플레이스로 가세요** ← 언매칭
5. **예약 1건 들어오면 보일러가 알아서 켜집니다 — 무인매장 난방 자동화하는 법**

---

## 본문

### 솔직히 말할게요. 그거 "원격제어" 아니라 "원격노동"입니다

겨울 아침 7시. 아직 이불 속인데 알람이 울립니다. 오늘 10시 파티룸 예약이 잡혀 있죠. 지금 안 일어나서 보일러를 안 켜두면, 손님이 도착했을 때 방은 냉골입니다. 결국 부스스 일어나 차 끌고 매장 가서 보일러 켜고 돌아옵니다.

더 황당한 건 그 다음입니다. 전날 밤 손님이 퇴실했는데, **보일러가 밤새 켜져 있었어요.** 빈 방을 여덟 시간 동안 데운 거죠. 가스비는 가스비대로 새고, "혹시 과열나는 거 아냐?" 불안은 불안대로.

간판은 "무인"인데, 정작 사장님이 **24시간 난방 당번**입니다.

이거, 진짜 무인 매장 맞으세요? 한 번 짚고 갑시다.

### 무인매장에서 보일러가 골치인 진짜 이유

무인 파티룸·무인카페·무인 스튜디오를 운영하면서 보일러 때문에 반복되는 일, 대충 이렇습니다.

- **빈 방 난방비 누수**: 퇴실 후 보일러가 계속 돌아갑니다. 아무도 없는 방을 데우는 시간이 길어질수록 가스비는 그대로 쌓입니다.
- **냉골 입실 → 별점 하락**: 겨울에 예약 시간 딱 맞춰 켜봐야 늦습니다. 적정 온도까지 올리려면 **30분~1시간 전엔 미리 돌려야** 하는데, 이걸 매번 손으로 챙기긴 어렵죠.
- **사장님이 갈려 나감**: 예약 건마다 켜고 끄는 걸 손으로 하면, 지점 2~3곳만 돼도 관리 부담이 폭발합니다.
- **안전 불안**: 장시간 무인 가동 중에 무슨 일 생기면? 원격으로 들여다볼 수단이 없으면 알 방법이 없습니다.

숫자로도 분명합니다. [기계설비신문의 지역난방 자동제어 분석](http://www.kmecnews.co.kr/news/articleView.html?idxno=32492)에 따르면, 자동제어를 결합한 난방 시스템은 에너지 소비를 **약 18~20% 절감**합니다. 사용 시간이 들쭉날쭉한 무인매장은 절감 폭이 더 큽니다.

그래서 무인매장에 보일러 원격제어는 "있으면 편한 기능"이 아니라 **돈이 새는 구멍을 막는 인프라**입니다. 문제는 *어떤 방식*으로 하느냐죠. 크게 두 갈래입니다.

### 1번 길: 보일러 회사 전용 컨트롤러 — 가정엔 OK, 무인매장엔?

경동나비엔·귀뚜라미·린나이. 국내 대표 3사 모두 자사 보일러 전용 IoT 컨트롤러와 앱을 줍니다. 기존 온도조절기를 와이파이 모델로 바꾸면 스마트폰으로 보일러를 켜고 끌 수 있죠.

- **경동나비엔 '나비엔 스마트'** — 난방·온수·외출 모드, 반복 예약, 실시간 가스 사용량, 구글 홈 음성 제어까지.
- **귀뚜라미 'IoT 매니저'** — 모드 설정, 24시간 예약, 고장 자가진단 알림, 이용 패턴 학습 스케줄 제안.
- **린나이 '스마트링크'** — GPS 귀가 알림, 기상 모드, 스마트 예약.

스펙만 보면 그럴듯합니다. **그런데 실사용자들 얘기는 좀 다릅니다.**

[클리앙의 한 귀뚜라미 사용기](https://www.clien.net/service/board/use/15789678)에서는 핵심 기능인 원격제어가 **몇 달간 먹통**이 됐다는 경험이 올라왔습니다. 서버가 주기적으로 다운되고, 앱 글씨가 화면 밖으로 잘리고, 온도조절기 두 대를 한 앱에서 같이 못 본다는 문제까지요.

[또 다른 클리앙 글](https://www.clien.net/service/board/park/14310557)의 댓글엔 더 뼈아픈 지적이 있습니다. **"왜 내 집 보일러를 제어하는데 린나이 서버를 거쳐서 처리하게 했는지 이해가 안 된다"**는 거죠. 제조사 서버를 경유하는 구조라 반응이 느리고, **서버가 죽으면 원격제어 자체가 불가능**합니다. 더 무서운 건, 제조사가 그 모델 IoT 서비스를 종료하면 멀쩡한 하드웨어를 두고도 원격제어를 못 쓰게 된다는 점. 경동나비엔이 옛 '스마트톡' 앱을 **2024년 9월 2일자로 종료**한 게 딱 그 사례입니다.

> 한 줄 요약: 내 매장 보일러인데, 켜고 끄는 권한이 보일러 회사 서버 사정에 달려 있다.

### "삼성 스마트싱스 연동되잖아요?" — 되긴 되는데, 믿고 맡길 수준이 아닙니다

세 회사 모두 삼성 [SmartThings](https://www.samsung.com/sec/smartthings/) 연동을 공식 지원합니다. SmartThings는 [API와 Webhook](https://developer.smartthings.com/docs/api/public)을 제공하니, 이론상 외부 시스템에서 보일러를 자동 제어할 수 있죠. 그런데 현장에선 한계가 뚜렷합니다.

- **연결이 불안정** — 삼성 커뮤니티엔 나비엔·귀뚜라미·린나이 모두 "기기 등록이 안 된다 / 연결이 끊긴다 / 어느 날부터 안 된다"는 사례가 반복됩니다.
- **제어가 반쪽** — 연동돼도 대부분 **전원 ON/OFF 수준**. 린나이는 공식적으로 "전원 on/off와 최근 조절 이력 조회"만 된다고 안내합니다. 온도 설정·모드 전환은 안 돼요.
- **지원 모델 제한** — 린나이는 단독 온도조절기 3개 모델만 되고, 각방 조절기는 제외.
- **결국 또 클라우드 의존** — SmartThings 클라우드를 한 번 더 거치니, 거기서 장애 나면 똑같이 멈춥니다.

가정에서 가끔 끄고 켜는 용도면 괜찮습니다. 하지만 **하루에 예약 여러 건을 처리하는 무인매장**의 자동화 인프라로 쓰기엔 부족합니다.

### 2번 길: 시하스(SiHAS) — 보일러 브랜드를 안 가립니다

여기서 갈립니다. 시하스(SiHAS, ㈜신아시스템)의 핵심은 한 문장입니다. **보일러 브랜드가 뭐든 작동하는 범용 컨트롤러.**

대표 제품 [BCM-300-W 시즌2](https://sihas.co.kr/product/bcm_300_s)는 경동나비엔·귀뚜라미·대성·린나이·디맥스 등 국내 주요 브랜드와 호환됩니다. 기존 온도조절기를 BCM-300으로 바꾸기만 하면, 브랜드와 상관없이 **시하스 앱 하나로** 다 제어돼요. 가격은 **정가 10만 원**. 인기라 품절·재입고가 잦으니 알림 신청해 두는 게 좋습니다.

설치도 부담 없습니다. [클리앙의 설치 사용기](https://www.clien.net/service/board/use/16979357)에선 "나사 풀고 배선만 교체하는 수준"으로 **10~30분 만에 자가 설치**했다는 후기가 있습니다. 앞서 먹통을 겪었던 귀뚜라미 사용자도 시하스로 갈아탄 뒤 *"자가설치 10분, 상담 친절, 앱 업데이트도 자주 해주는 신세계"*라고 평했고요.

무인매장 관점에서 시하스가 유리한 진짜 이유는 둘입니다.

- **다중 지점 통합** — A지점 나비엔, B지점 귀뚜라미여도 BCM-300 하나로 똑같이 제어. 지점마다 다른 앱 깔 필요 없습니다.
- **확장성** — 보일러뿐 아니라 조명 스위치·콘센트·가스밸브 차단기 같은 다른 IoT 기기까지 같은 생태계에서 관리. 게다가 SmartThings·구글 홈·KT 기가지니·네이버 클로바와 공식 연동돼서, **특정 회사에 종속되지 않습니다.**

> 지역난방(지멘스 RDD 통신형) 매장이라면 시하스 HCM-300 시리즈로 같은 걸 할 수 있습니다.

### 전용 컨트롤러 vs 시하스, 한눈에 비교

무인매장 사장님이 따져야 할 기준은 딱 둘입니다. **"예약 시스템이랑 연동되나"**, 그리고 **"여러 지점을 하나로 관리되나"**.

| 비교 항목 | 보일러 전용 컨트롤러 | 시하스 BCM-300 |
|---|---|---|
| 호환 보일러 | 자사 브랜드만 (나비엔→나비엔) | 경동·귀뚜라미·대성·린나이·디맥스 범용 |
| 관리 앱 | 제조사별 따로 (지점마다 다름) | 시하스 앱 하나로 통합 |
| 앱 안정성 | 장기간 먹통 사례 다수 | 잦은 업데이트, 안정적이란 평 |
| 서버 구조 | 제조사 서버 경유 → 단종·종료 리스크 | 시하스 자체 클라우드 |
| 외부 연동 | 자사 앱 위주 (스마트싱스 불안정) | 스마트싱스·구글홈·기가지니·클로바 |
| 예약 기능 | 매일/요일 고정 시간 반복뿐 | 스케줄 + 외부 연동 가능 |
| **실시간 예약 연동** | 별도 개발 필요 | **오토플레이스 연동으로 즉시** |
| 다른 IoT 기기 | 일부만 | 조명·콘센트·가스밸브 통합 |
| 다중 지점 관리 | 지점별 앱 분리 | 한 앱에서 전체 |
| 셀프 설치 | 설치기사 의존 | 10~30분 자가설치 사례 다수 |
| 가격대 | 5만~15만 원 (모델별) | 10만 원 (BCM-300-W 시즌2) |

가정에서 보일러 한 대만 원격으로 만지는 거라면, 해당 브랜드 전용 앱도 선택지입니다. 다만 앱 안정성과 서버 종속 리스크는 감수해야 하죠.

**하지만 무인매장은 얘기가 다릅니다.** 예약에 맞춰 보일러가 알아서 켜지고 꺼져야 하고, 보일러뿐 아니라 조명·출입·전력까지 한 시스템에서 굴러가야 합니다. 이걸 SmartThings API로 직접 구현하려면 개발자가 붙어야 하고, 연결 안정성도 장담 못 합니다. 범용으로 호환되면서 무인매장 전문 플랫폼과 바로 붙는 시하스가 합리적인 이유입니다.

### 진짜 핵심: 보일러를 "예약"에 묶는 것 — 오토플레이스

여기서 한 단계가 남았습니다. 시하스로 원격제어가 되더라도, **예약 시간에 맞춰 보일러가 알아서 움직이게** 만드는 건 또 다른 문제죠. 이걸 해결하는 게 [오토플레이스(AutoPlace)](https://autoplace.openiot.app)입니다.

오토플레이스는 오픈아이오티(OpenIoT)가 만든 무인 공간 운영 자동화 SaaS입니다. 슬로건 그대로예요. **"'편리한 IoT'는 필요없다. 돈 벌어주는 IoT."** 예약 → 결제 → 기기제어 → 정산까지 무인매장 운영 전체를 하나로 묶습니다. 현재 **설치 지점 10곳 이상**, 파티룸 현장에서 검증 중입니다.

작동 방식은 이렇습니다.

1. **예약 발생** — 손님이 파티룸을 예약합니다 (예: 오늘 14:00~17:00).
2. **사전 가동** — 시작 30분~1시간 전(설정 가능)에 보일러가 자동으로 켜집니다. 겨울엔 예열 시간을 더 길게.
3. **출입 연동** — 예약 시간에 맞춰 **1회용 도어락 비밀번호**가 자동 발급, 손님 입실.
4. **실시간 모니터링** — 대시보드에서 실내 온도·보일러 상태·재실 여부를 한눈에.
5. **자동 차단** — 종료 시각(또는 퇴실 확인)이 지나면 보일러·조명·콘센트가 같이 꺼집니다.

이 과정에서 사장님이 손대는 단계는 **없습니다.** 예약이 들어오는 순간부터 퇴실 후 기기 차단까지 전부 자동이에요.

### 실제 하루 시나리오 — 겨울 파티룸, 예약 2건

**오전 예약 A (10:00~13:00)**
- **09:00** — 시작 1시간 전, 보일러 자동 ON. 설정 22도.
- **09:30** — 1회용 비번 담긴 알림톡 자동 발송.
- **10:00** — 손님 입실. 실내 이미 21도, 쾌적. 조명도 자동 ON.
- **13:05** — 퇴실 확인되면 보일러·조명·콘센트 자동 OFF.

**오후 예약 B (15:00~18:00)**
- **13:05~14:00** — 보일러 꺼진 채. 빈 방 데우는 가스비 **0원**. 약 2시간 절약.
- **14:00** — 시작 1시간 전, 보일러 다시 ON.
- **15:00** — 손님 입실, 이미 따뜻함.
- **18:00** — 종료·퇴실 후 전체 자동 차단.

**사장님 개입: 없음.** 대시보드로 지켜볼 순 있지만, 직접 켜고 끌 일은 없습니다. 지점이 3곳이어도 한 대시보드에서 통합 관리되고, A는 나비엔 B는 귀뚜라미여도 시하스 BCM-300이면 똑같이 굴러갑니다.

### 자주 묻는 질문 (도입 전에 다들 걱정하는 것들)

**Q. 기존 보일러 온도조절기는 버려야 하나요?**
아닙니다. 시하스로 교체할 때 **기존 조절기는 꼭 보관**하세요. 보일러 AS 시 제조사가 기존 조절기 기준으로 점검합니다. 시하스 설치 후에도 보일러 자체 보증·AS는 그대로 유지됩니다.

**Q. 와이파이가 약한 매장도 되나요?**
시하스 BCM-300은 **전력선 통신(PLC)** 방식이라 보일러–컨트롤러 통신은 와이파이와 무관하게 안정적입니다. 단, 원격제어와 오토플레이스 연동엔 매장 인터넷이 필요한데, CCTV·결제 단말기용 인터넷이 이미 깔린 무인매장이 대부분이라 추가 공사가 필요한 경우는 드뭅니다.

**Q. 원격으로 켜두면 안전 문제는 없나요?**
가스보일러엔 과열방지·소화안전·정전안전 등 여러 안전장치가 내장돼 있고, 이것들은 원격제어와 **무관하게 항상 독립 작동**합니다. 원격제어는 온도조절기 역할을 대신할 뿐, 안전 메커니즘을 우회하는 게 아니에요. 오토플레이스 실시간 모니터링이 이상 온도·장시간 가동을 잡아 알림을 주니, 오히려 사전 대응이 쉬워집니다.

**Q. 여름엔요?**
같은 구조를 에어컨에도 씁니다. IoT 콘센트로 에어컨 전원을 잡으면 예약 시간에 맞춰 냉방을 자동으로 켜고 끌 수 있어요. 계절 관계없이 냉난방 자동화가 됩니다.

### 정리 — 자리를 비워도, 매장은 따뜻하게 돌아갑니다

무인매장 보일러의 핵심은 "앱으로 켜고 끄기"가 아닙니다. **예약에 맞춰 알아서 돌고, 출입·조명·전력과 한 몸으로 움직이는 것.** 거기에 진짜 가치가 있어요.

- **가정용**으로 보일러 한 대면 → 브랜드 전용 컨트롤러로 충분합니다. (앱 안정성·서버 종속은 감안)
- **무인매장**으로 예약 연동·다중 지점이 필요하면 → **시하스 범용 컨트롤러 + 오토플레이스** 조합이 답입니다.

도입 순서도 간단합니다.

1. **현장 파악** — 보일러 브랜드·모델, 매장 인터넷 확인
2. **하드웨어** — 시하스 BCM-300 교체 (+필요 시 콘센트·조명 등)
3. **오토플레이스 연동** — 공간 등록 → 기기 연결 → 예약 플랫폼 연동 → 자동화 룰 설정
4. **최적화** — 계절별 예열 시간 조정, 에너지 데이터 분석

새벽에 보일러 켜러 차 끌고 가는 일, 빈 방 데우느라 새는 가스비, 냉골 입실로 깎이는 별점. 이거 사장님이 안 해도 됩니다. 시스템이 대신합니다.

**무인매장 난방 자동화, 지금 바로 알아보기 → https://autoplace.openiot.app**

보일러 브랜드, 매장 수, 쓰고 계신 예약 플랫폼만 알려주시면 현장에 맞는 구성을 안내해 드립니다.
