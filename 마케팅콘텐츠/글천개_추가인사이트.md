# 글천개 추가 인사이트 (신규 대본 13편)

> 전제: 기존 플레이북(자청/이상한마케팅 51편)이 이미 담은 4사분면·표본이론·모텔이론·숫자의원칙·의심격파·U-I-S·유튜브4단계공식·가격설정·제안서원칙과 **중복되는 것은 전부 제외**했다. 아래는 글천개에서만 나오는, 그리고 openIoT B2B 단기현금(6개월) 리드젠을 실제로 바꾸는 인사이트만 추렸다.

---

## 0. 한 줄 요약

글천개의 모든 대본은 **"안 팔리는 이유는 실력이 아니라 '구매 순간의 선택지'에 안 들어가 있어서"** 하나로 수렴한다. 자청류가 '콘텐츠 품질·퍼널·전환 최적화'를 말한다면, 글천개는 **인지(기억) 점유 > 실력**, **상황 타겟 > 사람 타겟**, **소수 진성고객 > 대량 도달**, **구조 진단 > 노력**을 민다. openIoT처럼 "기술은 좋은데 아무도 모르는" B2B에 정확히 들어맞는다.

아래는 글천개에도 나오지만 **이미 플레이북에 있어 채택 안 함**:

| 버린 프레임 | 중복 사유 |

## 2. 글천개 신규 인사이트 12선 (채택)

### A. 진단·구조 계열 (단기현금에 가장 직결)

**① 매출 3분해 공식 — 변수는 단 하나**
- 매출 = `고객수 × 구매빈도 × 가격`. B2B로 번역하면 `인지된 의사결정자 수 × 재계약 빈도 × 계약단가`.
- 가격은 시장/RFP가, 재계약 빈도는 프로젝트 주기가 정하는 **상수**. 6개월 안에 통제 가능한 유일한 레버 = **"우리를 아는 의사결정자 수"**.
- → 기존 100개 후보를 더 쪼는 것보다 **인지된 후보 풀 자체를 200개로 넓히는 데 영업 시간을 배분**.

**② 3단계 매출 진단 (트래픽-이탈-전환) — "열심히 하지 말고 진단부터"**
- 매출 부진은 세 곳 중 하나만 막힌 것: **트래픽**(리드 자체 없음) / **이탈**(랜딩·제안서·데모에서 빠짐) / **전환**(미팅 후 클로징 실패).
- 세 군데 다 손대면 6개월을 날린다. **지금 막힌 한 곳만 진단해 그것만 뚫는다.**

| 구간 | openIoT 매핑 | 진단 신호 | 처방 |
|---|---|---|---|
| 트래픽 | 인바운드 리드·콘텐츠 도달 | 한 달 신규 문의 1~2건 미만 | 인지 노출(③④⑤) |
| 이탈 | 제안서·데모요청에서 이탈 | 미팅은 잡히나 검토 중 사라짐 | 후킹-본문-결론 3단 압축, 사례형 콘텐츠 |
| 전환 | 미팅 후 계약 | 다 보고도 타벤더 선정 | 믿음(레퍼런스 수치)+다음행동 버튼(⑪) |

**③ 전환 = 믿음(사례·후기·수치) × 구조(자동 다음행동)**
- "좋다면서 나한테 안 산다"면 **믿음이 없는 것**. 믿음은 얼굴/설명이 아니라 **구체 성과 수치(다운타임 X% 감소, 운영비 Y원 절감)**로만 생긴다.
- "다 보고도 행동 안 한다"면 **구조가 없는 것**. 미팅을 '검토해보세요'로 끝내지 말고 항상 **PoC 신청·견적 승인·파일럿 계약이라는 단일 다음 버튼**을 깔아둔다. 둘 중 하나만 없어도 전환은 0.

### B. 타겟·메시지 계열

**④ 사람이 아니라 '상황'을 타겟하라 (구매 트리거)**
- 인구통계 페르소나("제조업 사장님") 타겟은 필패. 사람은 안 변하고 **상황이 변하는 순간 구매 버튼이 눌린다.**
- openIoT 구매 트리거 매핑 → 그 동선에 미리 진열:

| 구매 트리거(상황) | openIoT가 떠올라야 할 순간 |
|---|---|
| 신규 설비·라인 증설 / 신공장 착공 | 레거시 SCADA 교체 검토 |
| 원격 모니터링·데이터 규제 대응 | ESG·안전 보고 의무화 |
| 기존 외주가 잠수/지연 | 다운타임 원인 사후 추적 한계 |
| 가동률·불량률 갑자기 악화 | 양산 앞두고 펌웨어·연동 막힘 |

- 콘텐츠·콜드메일을 **"~한 상황이 터졌을 때 이렇게 하라"** 형식으로 재구성.

**⑤ 레파토리 시장 + 라이트 유저가 금맥**
- 대부분 시장은 고객이 공부 안 하는 '레파토리 시장' → **가장 많이 노출된 자가 이긴다.** B2B도 의사결정자는 비교 공부할 시간이 없는 '라이트형'이 다수.
- **반(反)통념: 헤비 유저(끝없이 스펙 캐묻고 결국 갈아타는 잠재고객)에 영업 리소스 과투입 금지.** 응대 부담만 크고 단기 수주 확률 낮다.
- → 전담 영업은 **"이 정도면 충분"하며 빠르게 결정하는 라이트형(검증된 표준 요구사항을 가진 중견 운영조직)**에 집중.

**⑥ 정보 비대칭 게이트키퍼 — "왜 이 가격인가"를 한 줄로**
- 고객은 싼 외주와 우리 단가의 진짜 차이를 모른다. **"왜 싼 외주가 위험한지 / 우리 구조가 왜 이 가격인지"를 한 문장 비유로 설명**할 수 있으면 가격비교 테이블에서 빠져나온다(가격 비탄력화).
- → 영업 자료에 **'왜 이 가격인가' 게이트키퍼 설명 1줄씩** 박기.

### C. 콘텐츠·자산 계열

**⑦ 콘텐츠 3단계 레벨 — 신규 고객은 오직 3단계에서만 나온다**
- 1단계(스펙 설명)·2단계(일반론)은 아무리 올려도 매출 0. **3단계 = "대부분 모르는데 알면 바로 써먹는" 실전 정보.**
- **전문가 맹점 역전: 엔지니어가 "이건 너무 기본이라 고객도 알겠지" 넘기는 노하우가 정확히 99%가 모르는 3단계 정보.** (예: 센서 데이터 연동 시 90%가 놓치는 프로토콜 설정 하나)

**⑧ 사례형 콘텐츠가 매출의 본체 (어그로·포지셔닝·사례 중)**
- 어그로=클릭, 포지셔닝=인지, **사례=매출.** 단기현금이 목표면 조회수용이 아니라 **고객사 도입 Before→After 수치 사례에 자원 집중.** 전담 영업이 클로징에 그대로 들고 갈 레퍼런스가 최고 ROI.

**⑨ 손실 프레임 + 비교(공정성) 프레임 카피**
- 이득보다 손해가 더 무섭다 → **"도입 시 절감 X원"이 아니라 "미도입으로 매월 새고 있는 손실 X원"**으로 재구성.
- **"동종 경쟁사는 이미 운영비 X% 줄였는데 귀사는 여전히 구식"** 비교 프레임은 B2B 의사결정자의 '뒤처짐 공포'를 자극해 긴급성 생성.

### D. 퍼널·오퍼 계열

**⑩ 선별 초대 퍼널 (팔지 않고 파는 구조)**
- 무료 진단/세미나를 '아무나'가 아니라 **"이미 IoT 도입 의지가 있고 예산 단계인 기업"만 선별 초대** → 영업 리소스를 고전환 리드에 집중.
- openIoT 퍼널: **무료 진단 → 유상 PoC → 본계약** (전담 영업이 진단 참가자를 직접 클로징).

**⑪ 퍼주는 구조 + 회수 구조는 반드시 한 쌍**
- 무료 리드자석(IoT 도입 진단 문서·ROI 계산 템플릿)만 깔고 **회수 구조(연락처 수집→후속→유료전환)를 안 만들면 "열심히 했지만 멍청한 것".** 단기현금 목표에서 회수 없는 무료 배포는 비용만 된다.

**⑫ 메타 증명(도그푸딩) — "우리도 이렇게 써서 OO% 절감"**
- IoT 효과를 주장하려면 **자사 운영(사내 공정·물류·에너지 모니터링)에 자사 플랫폼을 적용한 before/after 수치를 공개**. 얕은 인지를 신뢰로 전환.
- → **오토플레이스(콘텐츠 40%)가 이미 '플랫폼 도그푸딩 증명' 역할로 재편된 것과 정확히 일치.** 이 각도를 의도적으로 강화.

## 3. 16억까지의 실전 교훈 (운영·리스크)

단기현금 목표에서 **현금 방어**에 직결되는 글천개 고유 교훈:

| 교훈 | 내용 | openIoT 적용 |
|---|---|---|
| **프로덕션 코스트 vs 유지비** | 망하는 1순위 = 초기 일회성 과지출 → 현금소진 → 조급 → 고객/마케팅에 '구걸' → 협상력 상실 | 과한 데모환경·대형부스·무리한 무상 PoC 개발 최소화. 자금 마르면 영업이 단가 깎으며 끌려간다 |
| **고정비 동결 룰** | 매출 신호 나와도 사무실·정규직 선제 투입 금지 | 수주 늘면 변동비(외주·계약직·파트너)로 흡수. 단위경제 검증 후에만 고정비 |
| **자발성 자산 법칙** | 외주 대행은 '연료'일 뿐 자산 안 남음. 내 손으로 만든 콘텐츠만 반복 재사용되는 영업 무기 | 마케팅 대행 의존 대신 전담 영업·엔지니어가 직접 사례·기술 Q&A 콘텐츠 생산 |
| **고객 자격 필터** | 사업 어려움의 90%는 '부적합 고객'. 진상 1건이 리소스 절반 잡아먹음 | ICP 스크리닝: 핸드홀딩만 요구하는 소규모, 단가만 후려치는 초대형사 거르고 'PoC 확장 예산·의사결정권 있는 중견'에 집중 |
| **환경결정론** | 실행력은 의지가 아니라 '현실적 목표 금액의 압박'이 결정 | 6개월 목표를 슬로건이 아니라 **월별 반드시 들어와야 하는 계약 금액**으로 환경화 (도달 가능한 수준으로 캘리브레이션) |

## 4. openIoT 단기현금 리드젠 — 무엇이 바뀌는가

### 채널
- 퍼포먼스 광고·대량 콜드콜 **선투입 금지**(B2B IoT 키워드 단가 높아 ROAS 붕괴). 광고는 콘텐츠로 인지를 깐 뒤 **리타게팅·검색광고로 마지막에 켜는 스위치**.
- 전담 영업 강점을 살려 **타깃 산업 의사결정자 100~300명 정밀 선정** → 이들 머릿속에 openIoT를 박는 집중 노출(레퍼런스 공유·기술 세미나·LinkedIn).

### 메시지
- 제안서/콜드메일 첫 줄을 **"저희 플랫폼은~"(내 이야기)에서 "귀사가 겪는 이 상황~"(상대·상황 이야기)**로 전면 전환.
- **'왜 이 가격인가' 게이트키퍼 1줄 + 손실 프레임("매월 새는 손실 X원") + 비교 프레임("경쟁사는 이미~")** 표준 탑재.

### 퍼널·오퍼
- **무료 진단 → 유상 PoC → 본계약**, 진단은 **선별 초대**, 모든 콘텐츠/제안서 말미에 **단일 CTA 버튼**.
- 리드자석은 **회수 구조와 반드시 한 쌍**.

### 콘텐츠
- openIoT 60%: **3단계 실전 정보 + 사례형(Before→After 수치)** 우선. '문제 20개 → 해결법 10개 → 이번 주 3개 발행' 스프린트.
- 오토플레이스 40%: **도그푸딩 메타 증명** 각도로 의도적 강화(자사 플랫폼 사내 적용 수치 공개).

> ⚠️ 주의: 글천개의 "노력하면 무조건 된다 / 안 되면 실력 문제"·"2~3개월이면 비탄력 전환" 같은 단정은 **B2B는 사이클이 길어 그대로 적용 무리.** 검증 기간을 길게 잡고, 단기현금은 '이미 인지된 핫리드 직접 영업'으로 메우는 **투트랙**이 현실적이다. 또한 글천개 본인이 '확신 화법 = 약간의 사기성'이라 자평한 만큼, **검증 불가한 과장은 B2B 레퍼런스 비즈니스에서 역효과** — "확신은 결과 약속에, 단서는 기술 조건에" 배분.
